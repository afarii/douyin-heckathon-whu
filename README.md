## 🧠 提示词：哈基米小游戏项目说明

### 项目背景

“哈基米”（中文网络梗）是一个在中文互联网流行的 **音频/文化模因热梗**，其原始来源与日本动画《赛马娘 Pretty Derby》中角色东海帝王哼唱的一段背景音乐片段相关。该词来自日语 “はちみ”（hachimi），原指 “蜂蜜 / 蜂蜜水”，后来因魔性重复音节以及在社交平台大量搭配萌宠（尤其是猫咪）短视频而被误读为 “可爱小动物 / 猫咪” 的代称。梗在不同圈层传播过程中含义多变、被不断再创作与语义扩散。([Wiktionary][1])

该梗热度高、传播广泛，既有鬼畜音频、抽象文化作品，也被用于宠物短视频背景音乐，是一个典型的 **具备声效识别 + 社交传播潜力的网络现象**。([static.nfapp.southcn.com][2])

## 🎮 游戏/Demo 功能简介

总体想法是：

1. **用户录制或上传音频**
2. **（后端）与“哈基米热梗原声”进行相似度比对**
3. **生成评分 + 梗化评价文本**
4. **在前端展示结果 + 可分享的海报/短文案/视频片段**

### 🎙 音频录制 & 处理（前端）

* 使用 **HTML5 Web Audio API + getUserMedia**

  * 实现浏览器内录音
  * 录制结束后直接生成 Blob
  * 可选显示录音波形或录音时长 UI
* 将音频文件 POST 上传到后端 REST API

---

### 📡 与后端对接

* 使用 **fetch** 或 **axios** 发起请求
* 后端接口设计（示例）：

  * `POST /api/upload` → 接收音频
  * 返回：`{ similarity: number, grade: string, comment: string }`
* 前端解析结果并展示评分、等级、文字评价

## 🧠 可后续集成点 — 与音频比对核心

> 后端比对逻辑暂不由前端负责，但前端应提前设计好可插拔的接口

后端核心比对可能使用音频特征提取 + 向量相似度算法等技术：

* 嵌入向量比对 + 余弦相似度
