# 哈基米相似度测试

## 项目简介

哈基米相似度测试是一个浏览器端音频检测小工具。用户可以先播放示例音频，再录制或上传自己的音频，由后端计算与示例素材的相似度，并生成分数、称号、评分理由和结算页。

项目当前采用后端实测评分：前端不会生成 Mock 分数，检测结果必须由 `/api/upload` 返回。示例音频使用 `基米素材/haqi.mp3`，提交时前端会把用户音频和示例音频转换为 WAV 后一起发送给后端。

主要体验包括：

* 播放示例音频并模仿。
* 录音或上传本地音频。
* 后端返回相似度分数、称号和评分理由。
* 结算页展示哈基米图片/动图、分数圆环、本机排行榜和分享图。
* 后端会限制普通说话、英语听力、音乐、噪声等无关音频的高分误判。

## 使用方法

本地运行：

```bash
python backend/server.py
```

默认访问：

```text
http://127.0.0.1:4173/
```

指定端口：

```powershell
$env:PORT=4180; python backend/server.py
```

使用流程：

1. 打开页面后，先播放示例音频。
2. 点击“开始录音”，允许浏览器使用麦克风。
3. 录完后点击检测，等待后端返回结果。
4. 也可以直接上传本地音频文件进行检测。
5. 进入结算页后，可以查看分数、称号、评分理由、本机排行榜，并保存或分享结算图。

部署注意：

* 录音依赖浏览器麦克风权限，线上环境建议使用 HTTPS。
* 前端请求的是相对路径 `/api/upload`，部署时建议前端和后端保持同源。
* 后端默认监听 `127.0.0.1:4173`，生产部署建议通过 Nginx/Caddy 等服务做 HTTPS 反向代理。
