# 哈基米相似度测试

一个面向浏览器的哈基米音频小游戏 Demo。用户可以录制或上传一段音频，前端会尝试提交到后端接口进行相似度检测；在后端尚未接入时，页面会自动使用本地 Mock 评分，保证演示流程完整可玩。

## 当前功能

* 浏览器内录音：基于 `getUserMedia` 和 `MediaRecorder` 采集音频
* 音频上传：支持用户选择本地音频文件
* 音频预览：录制或上传后可直接播放
* 哈基米检测：优先请求 `POST /api/upload`
* Mock 回退：接口不可用时自动生成本地评分和梗化文案
* 结算页面：检测后展示哈基米图片素材、相似度分数、称号和评价文案
* 分享结果：支持系统分享，浏览器不支持时复制分享文案
* 响应式界面：适配手机和桌面浏览器

## 本地运行

项目当前是零依赖静态页面，可以直接用任意静态服务器启动。

```bash
python -m http.server 4173 --bind 127.0.0.1
```

然后访问：

```text
http://127.0.0.1:4173/
```

> 录音能力依赖浏览器麦克风权限。建议通过 `localhost` 或 `127.0.0.1` 访问。

## 后端接口约定

前端会将音频通过 `FormData` 上传到：

```http
POST /api/upload
```

字段名：

```text
audio
```

期望返回：

```json
{
  "similarity": 88,
  "grade": "神级哈基米",
  "comment": "你登上哈基宇宙排行榜了！"
}
```

如果接口不存在、请求失败或返回非 `2xx`，前端会自动进入 Mock 模式。
