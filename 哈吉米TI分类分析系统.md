# 哈吉米TI（Hajimi Type Indicator）分类分析系统

> 版本：V2.0（内容体系重构版）  
> 文档定位：用于“梗文化内容分级 + 用户画像识别 + 社区运营决策”的统一标准

---

## 一、文档目标与适用范围

### 1.1 目标

本体系用于解决三个核心问题：

1. 如何评估一个“哈吉米相关内容/梗”的文化势能与传播价值。  
2. 如何识别不同用户在哈吉米文化中的参与类型与成长阶段。  
3. 如何将分类结果用于社区运营、内容推荐、活动激励与商业协同。

### 1.2 适用对象

- **内容侧**：短视频、音频、二创作品、话题事件、联名活动。  
- **用户侧**：普通观看者、内容创作者、传播者、社群核心成员。  
- **运营侧**：社区运营、内容运营、数据分析、品牌合作。

### 1.3 不适用场景

- 作为严肃心理学诊断工具（本体系不具备临床效力）。  
- 作为法律意义上的行为评判依据。  
- 脱离语境直接对个体贴“价值标签”。

---

## 二、核心概念与文化边界

### 2.1 关键术语

- **哈吉米**：由“はちみ（蜂蜜）”在网络语境中演化出的跨平台梗符号。  
- **耄耋宇宙**：围绕“圆头耄耋”事件形成的叙事共同体与二创生态。  
- **TI等级**：衡量内容或用户综合影响力的分级结果（S/A/B/C/D/F）。  
- **哈吉米MBTI**：基于互动行为设计的16型趣味人格框架（非心理诊断）。

### 2.2 文化边界原则

- **尊重动物与现实伦理**：反对伤害与鼓励伤害行为。  
- **允许多元表达**：原教旨、抽象派、商业派可并存。  
- **反对极端排他**：禁止以“圈层纯度”进行群体攻击。

## 三、TI评分模型（内容侧）

### 3.1 四维指标与权重

| 维度 | 权重 | 评估问题 |
|---|---:|---|
| 音频感染力（Audio Impact） | 30% | 是否具有记忆点、可模仿性与情绪驱动？ |
| 文化共鸣度（Cultural Resonance） | 25% | 是否承载共同语境、群体认同与叙事价值？ |
| 传播裂变力（Viral Potential） | 25% | 是否易二创、易传播、可跨圈层扩散？ |
| 社区归属感（Community Belonging） | 20% | 是否强化身份感、参与感与长期留存？ |

### 3.2 单维评分标准（1-5分）

| 分值 | 描述 |
|---:|---|
| 1 | 基本无特征，难以形成传播 |
| 2 | 有弱特征，仅限小范围 |
| 3 | 具备稳定可识别特征 |
| 4 | 特征突出，具备明显扩散能力 |
| 5 | 现象级特征，可形成文化符号 |

### 3.3 计算公式

- **标准分** = `Σ(维度得分 × 维度权重)` × 20  
- 标准分区间：0-100

### 3.4 TI等级映射

| TI等级 | 分数区间 | 解释 |
|---|---:|---|
| TI-S | 90-100 | 现象级符号，具“时代梗”潜力 |
| TI-A | 80-89 | 标杆级内容，长期可复用 |
| TI-B | 70-79 | 高质量圈层内容，具稳定受众 |
| TI-C | 60-69 | 有潜力内容，需运营放大 |
| TI-D | 50-59 | 跟风型/浅参与内容 |
| TI-F | 0-49 | 弱关联或不具文化连接 |

## 四、对象分层体系

### 4.1 内容对象分层

| 层级 | 定义 | 典型代表 |
|---|---|---|
| 源点型 | 梗源头或关键起点 | 原始音频/起源角色 |
| 叙事型 | 能形成故事与世界观 | 事件链、角色关系 |
| 玩法型 | 可复制可改编 | 鬼畜调音、混剪模板 |
| 商业型 | 与品牌或产品结合 | 联名活动、商业短片 |
| 消费型 | 仅跟随热点 | 普通刷梗内容 |

### 4.2 用户参与分层

| 层级 | 行为特征 | 运营目标 |
|---|---|---|
| 核心创作者 | 原创与高频二创 | 维护创作动力 |
| 深度传播者 | 高质量转发与解释 | 建立传播激励 |
| 社区贡献者 | 评论、答疑、组织活动 | 提升社群粘性 |
| 轻度参与者 | 点赞、收藏、偶发互动 | 引导向中度参与 |
| 圈外观察者 | 低兴趣、低理解 | 提供低门槛入口 |

### 5.3 使用原则

- 仅用于运营分群、内容匹配、活动设计。  
- 不用于人格优劣排序。  
- 支持动态变化（用户类型可随行为改变）。

## 六、称号系统（荣誉层）

### 6.1 设计目标

- 提供可见成长路径。  
- 增强社区身份认同。  
- 激励优质创作和持续参与。

### 6.2 命名结构

`[稀有度前缀] [TI等级] · [核心定位] · [文化标签] · [后缀]`

示例：`💎 TI-S · 圆哈镇之主 · 抽象大师 · 耄耋`

### 6.3 稀有度体系

| 稀有度 | 概率建议 | 适用范围 |
|---|---:|---|
| SSR | 0.5% | 文化符号、年度级贡献 |
| SR | 3% | 高影响力创作与传播 |
| R | 12% | 持续贡献型成员 |
| N | 84.5% | 常规参与成员 |

### 6.4 升降级机制（建议）

- 升级依据：内容质量、互动质量、持续活跃、社区贡献。  
- 降级触发：长期不活跃、违规行为、恶意扰动。  
- 保护机制：短期波动不降级，按季度复核。

## 七、测评流程与数据实现

### 7.1 流程总览

1. **采集**：行为日志、互动数据、内容标签、问卷结果。  
2. **清洗**：去重、异常过滤、账号合并。  
3. **评分**：内容TI评分 + 用户MBTI维度计算。  
4. **分层**：等级归类、称号授予、运营标签写回。  
5. **复核**：人工抽检 + 争议案例复判。

### 7.2 推荐数据字段

| 字段 | 说明 |
|---|---|
| content_id / user_id | 内容与用户主键 |
| play, like, comment, share | 基础互动指标 |
| save_rate, completion_rate | 质量与沉浸指标 |
| remix_count | 二创扩散指标 |
| community_actions | 社区贡献行为 |
| survey_vector | 测评问卷向量 |

### 7.3 问卷计分规则（简版）

- 每题二选一，分别映射到一对维度。  
- 每个维度建议设置3-5题。  
- 维度分差低于阈值时可标记“中性倾向”。

## 八、运营应用场景

### 8.1 社区运营

- 对 TI-S/TI-A 用户开放“导师/评审/共创”权限。  
- 对 TI-C/TI-D 用户推送入门任务和轻量激励。  
- 为不同MBTI群体配置差异化话题包。

### 8.2 内容分发

- 高感染力内容用于拉新。  
- 高共鸣内容用于留存。  
- 高裂变内容用于活动节点冲刺。  
- 高归属内容用于社群沉淀。

### 8.3 商业协同

- 联名前先做“文化稀释风险评估”。  
- 商业内容须保留社区语境锚点。  
- 商业化结果纳入 TI-C/TI-A 转化分析。

---

## 九、风险控制与伦理规范

- 禁止基于标签进行歧视、排斥或人身攻击。  
- 对未成年人群体设置推荐与互动保护策略。  
- 对争议梗设立人工审核和申诉机制。  
- 对外传播需注明“趣味人格系统，非医学/心理诊断”。
