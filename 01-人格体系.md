## 一、设计理念

### 1.1 系统概述

哈基米TI（Hajimi Type Indicator）是一个基于**音频物理特征**的人格分类系统，参考 MBTI 的四维度模型和 SBTI 的娱乐化风格设计。

**核心原则：**
- 每个维度基于真实的音频物理特征判定，不是问卷
- 命名融合猫咪行为学 + 网络梗文化 + 自嘲幽默
- 共 **16种常规人格 + 2种隐藏人格**（总计18种）
- 每种人格附带完整的趣味分析文案和专属"哈气画像"

### 1.2 与 MBTI/SBTI 的对比

| 对比项 | MBTI | SBTI | 哈基米TI |
|--------|------|------|----------|
| 判定方式 | 问卷（93题） | 问卷（15维度） | 音频物理特征分析 |
| 维度数 | 4维度 | 15维度 | 4维度 |
| 类型数 | 16种 | 多种组合 | 16常规 + 2隐藏 |
| 输入方式 | 自评问卷 | 自评问卷 | 录制哈气音频 |
| 娱乐性 | 低 | 高 | 极高 |
| 科学性 | 中（争议） | 无 | 无（纯娱乐） |

### 2.2 维度一：H/S — 哈气力（Hiss）vs 静默力（Silence）

**核心问题：你的哈气有多大攻击性？**

#### H（Hiss）— 哈气型

| 项目 | 内容 |
|------|------|
| 代号 | H |
| 含义 | 攻击性强、表达直接、不藏着掖着 |
| 猫咪类比 | 耄耋本耄——上来就是飞机耳加嘶哈，绝不跟你废话 |
| 判定条件 | 平均响度 > 65dB **且** 频率峰值 > 4000Hz |
| 边界条件 | 如果不满足H也不满足S，则 HIS强度 > 5 判定为H，≤5 判定为S |
| 关联音频特征 | avgDB（平均响度）、peakFreq（频率峰值） |

**H型人格文案：**
> "你的哈气就像耄耋本耄——上来就是飞机耳加嘶哈，绝不跟你废话。攻击性拉满，社交场上你是那个'别惹我'的存在。你的哈气不需要翻译，对方一听就知道：退后。"

#### S（Silence）— 静默型

| 项目 | 内容 |
|------|------|
| 代号 | S |
| 含义 | 内敛克制、蓄力型、一鸣惊人 |
| 猫咪类比 | 一只蹲在暗处的猫，平时安安静静，一旦出手就是暴击 |
| 判定条件 | 平均响度 < 45dB **或** 录制时长中静音段 > 60% |
| 边界条件 | 同上 |
| 关联音频特征 | avgDB（平均响度）、silenceRatio（静音占比） |

**S型人格文案：**
> "你不是不会哈气，你只是在等最佳时机。就像一只蹲在暗处的猫，平时安安静静，一旦出手就是暴击。你的哈气可能不大声，但每一声都恰到好处——精准、克制、致命。"

### 2.3 维度二：L/P — 持久型（Long）vs 脉冲型（Pulse）

**核心问题：你的哈气是持久战还是闪电战？**

#### L（Long）— 持久型

| 项目 | 内容 |
|------|------|
| 代号 | L |
| 含义 | 耐力惊人、稳定输出、持久战选手 |
| 猫咪类比 | 一只被踩了尾巴还能哈气30秒的猫——耐力惊人 |
| 判定条件 | 哈气持续时间 > 1.5s **且** 音量波动 < 15% |
| 边界条件 | 如果不满足L也不满足P，则基于 duration 中值（1.0s）判定，>1.0s为L |
| 关联音频特征 | activeDuration（有效时长）、volumeVariance（音量方差） |

**L型人格文案：**
> "你的哈气像永动机——别人哈三秒就喘，你能哈到对方怀疑人生。持久战是你的主场，消耗战没在怕的。你不是在哈气，你是在做有氧运动。"

#### P（Pulse）— 脉冲型

| 项目 | 内容 |
|------|------|
| 代号 | P |
| 含义 | 爆发力强、节奏多变、短平快 |
| 猫咪类比 | 一只连续短促哈气的猫——哒哒哒哒，像连发弹夹 |
| 判定条件 | 哈气持续时间 < 0.8s **或** 音量波动 > 30% |
| 边界条件 | 同上 |
| 关联音频特征 | activeDuration（有效时长）、volumeVariance（音量方差） |

**P型人格文案：**
> "你的哈气像连发弹夹——短促、密集、每一发都精准打击。别人还在蓄力，你已经完成三轮输出。你的战术是：打一枪换一个地方，让对方永远摸不清你的节奏。"

### 2.4 维度三：T/F — 高音型（Treble）vs 低沉型（Flat）

**核心问题：你的哈气是尖锐刺耳还是厚重压抑？**

#### T（Treble）— 高音型

| 项目 | 内容 |
|------|------|
| 代号 | T |
| 含义 | 尖锐刺耳、穿透力强、存在感拉满 |
| 猫咪类比 | 一只发出超高频嘶哈的猫——方圆十米都能感受到 |
| 判定条件 | 主频率 > 5000Hz **或** 频率峰值 > 7000Hz |
| 边界条件 | 如果不满足T也不满足F，则基于 dominantFreq 中值（4000Hz）判定，>4000Hz为T |
| 关联音频特征 | dominantFreq（主频率）、peakFreq（峰值频率） |

**T型人格文案：**
> "你的哈气自带穿透buff——方圆十米内的生物都能感受到你的不满。声音尖锐但不空洞，每一赫兹都是实打实的愤怒。你的哈气频率已经进入了'猫科动物听觉最敏感区域'，这是进化级别的威慑。"

#### F（Flat）— 低沉型

| 项目 | 内容 |
|------|------|
| 代号 | F |
| 含义 | 厚重压抑、威慑力强、气场型 |
| 猫咪类比 | 一只从胸腔深处发出低频轰鸣的猫——不刺耳但让人后背发凉 |
| 判定条件 | 主频率 < 3000Hz **且** 低频能量占比 > 40% |
| 边界条件 | 同上 |
| 关联音频特征 | dominantFreq（主频率）、lowFreqRatio（低频能量占比） |

**F型人格文案：**
> "你的哈气像低音炮——不刺耳，但让人后背发凉。那种从胸腔深处涌出的低频震动，是'我认真了'的终极信号。你不是在哈气，你是在发出次声波——对方可能听不清，但身体已经本能地后退了。"

### 2.5 维度四：C/R — 混沌型（Chaos）vs 节奏型（Rhythm）

**核心问题：你的哈气有规律吗？**

#### C（Chaos）— 混沌型

| 项目 | 内容 |
|------|------|
| 代号 | C |
| 含义 | 不按套路出牌、随机性强、不可预测 |
| 猫咪类比 | 一只哈气时音调忽高忽低、毫无规律的猫——抽象派 |
| 判定条件 | 音调变化率 > 500Hz/s **或** 频率方差 > 1500Hz² |
| 边界条件 | 如果不满足C也不满足R，则基于 pitchChangeRate 中值（350Hz/s）判定，>350Hz/s为C |
| 关联音频特征 | pitchChangeRate（音调变化率）、freqVariance（频率方差） |

**C型人格文案：**
> "你的哈气是抽象派艺术——没有规律、没有章法、甚至没有逻辑。但正是这种不可预测性，让对手永远猜不到你下一步。你的哈气频率变化之快，连频谱分析仪都跟不上。"

#### R（Rhythm）— 节奏型

| 项目 | 内容 |
|------|------|
| 代号 | R |
| 含义 | 有规律、可控性强、技术流 |
| 猫咪类比 | 一只哈气像节拍器一样精准的猫——教科书级别 |
| 判定条件 | 音调变化率 < 200Hz/s **且** 频率方差 < 500Hz² |
| 边界条件 | 同上 |
| 关联音频特征 | pitchChangeRate（音调变化率）、freqVariance（频率方差） |

**R型人格文案：**
> "你的哈气是教科书级别的——频率稳定、节奏均匀、像节拍器一样精准。你不是在哈气，你是在演奏一首'退后'交响曲。你的控制力让对手感到不安：这么稳定的哈气，说明你完全在掌控之中。"

## 五、人格判定算法

### 5.1 判定流程

```
输入：FeatureVector 音频特征向量

第一步：隐藏人格检测（优先）
  ├── 检测 MEOOOW：dominantFreq < 1500 && lowFreqDuration > 0.5
  │   └── 如果满足 → 返回 "MEOOOW" + 匹配度
  ├── 检测 SILENT：avgDB < 20 || duration < 0.3
  │   └── 如果满足 → 返回 "SILENT" + 匹配度
  └── 都不满足 → 进入第二步

第二步：四维度判定
  ├── 维度一 H/S：
  │   ├── avgDB > 65 && peakFreq > 4000 → H
  │   ├── avgDB < 45 || silenceRatio > 0.6 → S
  │   └── else → HIS > 5 ? H : S
  │
  ├── 维度二 L/P：
  │   ├── duration > 1.5 && volumeVariance < 0.15 → L
  │   ├── duration < 0.8 || volumeVariance > 0.3 → P
  │   └── else → duration > 1.0 ? L : P
  │
  ├── 维度三 T/F：
  │   ├── peakFreq > 5000 || dominantFreq > 7000 → T
  │   ├── dominantFreq < 3000 && lowFreqRatio > 0.4 → F
  │   └── else → dominantFreq > 4000 ? T : F
  │
  └── 维度四 C/R：
      ├── pitchChangeRate > 500 || freqVariance > 1500 → C
      ├── pitchChangeRate < 200 && freqVariance < 500 → R
      └── else → pitchChangeRate > 350 ? C : R

第三步：组合人格代号
  输出：四字母代号（如 "HLTC"）+ 各维度匹配度百分比
```

### 5.2 匹配度计算

每个维度的匹配度表示该维度判定结果的"确信程度"：

```
H/S 维度匹配度：
  如果明确满足H或S条件 → 85-95%
  如果通过HIS边界判定 → 55-75%

L/P 维度匹配度：
  如果明确满足L或P条件 → 85-95%
  如果通过duration中值判定 → 55-75%

T/F 维度匹配度：
  如果明确满足T或F条件 → 85-95%
  如果通过dominantFreq中值判定 → 55-75%

C/R 维度匹配度：
  如果明确满足C或R条件 → 85-95%
  如果通过pitchChangeRate中值判定 → 55-75%

综合匹配度 = 四个维度匹配度的平均值
```
