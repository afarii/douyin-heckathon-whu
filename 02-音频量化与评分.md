## 一、音频特征提取总览

### 1.1 特征分类

音频特征分为三大类：**频率特征**、**响度特征**、**时间特征**，外加一个**变化特征**。

| 类别 | 特征名称 | 英文标识 | 单位 | 说明 |
|------|----------|----------|------|------|
| 频率 | 主频率 | dominantFreq | Hz | 能量最大的频率分量 |
| 频率 | 峰值频率 | peakFreq | Hz | 频谱中的最高频率峰值 |
| 频率 | 平均频率 | avgFreq | Hz | 加权平均频率 |
| 频率 | 频率范围 | freqRange | Hz | 有效频率范围（最高-最低有意义频率） |
| 频率 | 音调变化率 | pitchChangeRate | Hz/s | 相邻帧之间频率差的时间导数 |
| 频率 | 频率方差 | freqVariance | Hz² | 频率稳定性的度量 |
| 频率 | 频谱质心 | spectralCentroid | Hz | 声音"明亮度"指标 |
| 频率 | 低频能量占比 | lowFreqRatio | 0-1 | <2000Hz频段能量/总能量 |
| 响度 | 平均响度 | avgDB | dB | 整个哈气段的RMS平均值 |
| 响度 | 峰值响度 | peakDB | dB | 最大瞬时响度 |
| 响度 | 最低有效响度 | minDB | dB | 哈气段中的最低有效响度 |
| 响度 | 动态范围 | dbRange | dB | peakDB - minDB |
| 响度 | 音量方差 | volumeVariance | 0-1 | 音量稳定性的度量（归一化值） |
| 响度 | RMS能量 | rmsEnergy | 0-1 | 整体能量水平（归一化值） |
| 时间 | 总时长 | duration | s | 从录制开始到结束的总时间 |
| 时间 | 有效时长 | activeDuration | s | 去除首尾静音后的实际哈气时长 |
| 时间 | 静音占比 | silenceRatio | 0-1 | 静音段时长 / 总时长 |
| 时间 | 起音时间 | attackTime | s | 从静音到最大响度的上升时间 |
| 时间 | 衰减时间 | decayTime | s | 从最大响度到静音的下降时间 |
| 时间 | 持续水平 | sustainLevel | dB | 稳态段的平均响度 |
| 变化 | 音调变化率 | pitchChangeRate | Hz/s | （与频率特征共用） |
| 变化 | 频率方差 | freqVariance | Hz² | （与频率特征共用） |
| 变化 | 音量方差 | volumeVariance | 0-1 | （与响度特征共用） |

### 1.2 技术实现参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 采样率 | 44100 Hz | 统一采样率，确保分析一致性 |
| FFT大小 | 4096 | 频率分辨率约 10.7Hz @ 44100Hz |
| 频率分辨率 | ~10.7 Hz | 44100 / 4096 |
| 平滑系数 | 0.3 | 较低平滑度以捕捉快速变化 |
| 最小分贝 | -90 dB | AnalyserNode.minDecibels |
| 最大分贝 | -10 dB | AnalyserNode.maxDecibels |
| 静音阈值 | RMS < 0.01 或 dB < 25dB | 连续帧低于此值视为静音 |
| 录制时长限制 | 1-5 秒 | 过短无法提取特征，过长影响体验 |
| 频率分析范围 | 20 - 20000 Hz | 人耳可听范围 |

## 二、频率（音调）量化标准

### 2.1 频率等级划分

| 等级 | 名称 | 频率范围 | 猫咪行为类比 | 得分范围 | 视觉颜色 |
|------|------|----------|-------------|----------|----------|
| F1 | 低频嗡鸣 | < 1500 Hz | 猫咪低吼/咕噜声 | 1-2分 | 深蓝 #00008B |
| F2 | 中低频 | 1500 - 3000 Hz | 不满的咕噜声 | 3-4分 | 蓝色 #0000FF |
| F3 | 中频哈气 | 3000 - 5000 Hz | 标准猫咪哈气 | 5-6分 | 绿色 #00FF00 |
| F4 | 高频警告 | 5000 - 7000 Hz | 强烈警告哈气 | 7-8分 | 橙色 #FFA500 |
| F5 | 超高频嘶哈 | 7000 - 10000 Hz | 极端威胁嘶哈 | 9-10分 | 红色 #FF0000 |

### 2.2 频率等级详细描述

#### F1 — 低频嗡鸣（< 1500 Hz）

> 你的哈气频率极低，像猫咪的低吼或咕噜声。这种频率通常不会让人感到威胁，更像是一种"不满的低语"。在猫科动物行为学中，低频声音多用于近距离沟通，传达的信息是"我不太高兴"而不是"退后"。

**典型场景：** 猫咪被轻微打扰时发出的低沉咕噜声、吃饱后不想被摸时的低频抗议。

#### F2 — 中低频（1500 - 3000 Hz）

> 你的哈气频率处于中低区间，像一只不太高兴的猫发出的咕噜声。这种频率已经可以明确传达"不满"的信息，但还不到"威胁"的程度。对方能听出你在哈气，但可能不会太当回事。

**典型场景：** 猫咪看到不认识的人时的警惕性哈气、被摸了不喜欢的地方时的抗议。

#### F3 — 中频哈气（3000 - 5000 Hz）

> 你的哈气频率处于标准区间，这是最典型的猫咪哈气频率范围。声音清晰可辨，传达的信息明确："我不高兴，请退后"。大多数家猫的哈气频率都在这个范围内。

**典型场景：** 猫咪被陌生人接近时的标准哈气、领地受到轻微侵犯时的警告。

#### F4 — 高频警告（5000 - 7000 Hz）

> 你的哈气频率偏高，进入了"强烈警告"区间。这种频率的哈气通常伴随着飞机耳和弓背，传达的信息是"我真的不高兴了，再不走我要动手了"。在猫科动物行为学中，高频哈气是攻击前的最后警告。

**典型场景：** 猫咪被逼到角落时的强烈哈气、看到天敌（如狗）时的警告。

#### F5 — 超高频嘶哈（7000 - 10000 Hz）

> 你的哈气频率极高，进入了"极端威胁"区间。研究表明，猫咪哈气的频率范围在9.5-10.5kHz之间，正好是大多数哺乳动物听觉最敏感的区域。你的哈气频率已经接近这个"生物武器"级别，对方的身体会本能地产生恐惧反应。

**典型场景：** 猫咪暴怒状态下的嘶哈、极度恐惧时的防御性尖叫。

### 2.3 频率得分计算公式

```
freqScore = clamp((dominantFreq - 1000) / 9000 * 10, 0, 10)

其中：
- dominantFreq: 主频率 (Hz)
- 1000: 起始频率偏移（低于1000Hz得0分）
- 9000: 频率范围跨度（1000Hz到10000Hz映射到0-10分）
- clamp: 将结果限制在0-10范围内
```

**示例计算：**
| dominantFreq | 计算过程 | freqScore |
|-------------|----------|-----------|
| 800 Hz | (800-1000)/9000*10 = -0.22 → 0 | 0 |
| 1500 Hz | (1500-1000)/9000*10 = 0.56 | 0.6 |
| 3000 Hz | (3000-1000)/9000*10 = 2.22 | 2.2 |
| 5000 Hz | (5000-1000)/9000*10 = 4.44 | 4.4 |
| 7000 Hz | (7000-1000)/9000*10 = 6.67 | 6.7 |
| 8500 Hz | (8500-1000)/9000*10 = 8.33 | 8.3 |
| 10000 Hz | (10000-1000)/9000*10 = 10.0 | 10.0 |

## 三、响度（分贝）量化标准

### 3.1 响度等级划分

| 等级 | 名称 | 平均响度范围 | 猫咪行为类比 | 得分范围 | 视觉颜色 |
|------|------|-------------|-------------|----------|----------|
| D1 | 气若游丝 | < 30 dB | 睡梦中呼吸 | 1-2分 | 浅灰 #D3D3D3 |
| D2 | 轻声细语 | 30 - 45 dB | 安静环境下的哈气 | 3-4分 | 银色 #C0C0C0 |
| D3 | 正常哈气 | 45 - 60 dB | 日常对话级别 | 5-6分 | 金色 #FFD700 |
| D4 | 大声警告 | 60 - 75 dB | 吵闹环境下的哈气 | 7-8分 | 橙色 #FF8C00 |
| D5 | 震耳欲聋 | 75 - 100 dB | 暴怒嘶吼 | 9-10分 | 红色 #FF4500 |

### 3.2 响度等级详细描述

#### D1 — 气若游丝（< 30 dB）

> 你的哈气响度极低，像猫咪睡梦中的呼吸声。这种响度几乎不会引起任何注意，对方可能根本没意识到你在哈气。在哈气PK中，这相当于"空放技能"——动画播了，但没伤害。

**参考对比：** 安静的图书馆（30dB）、耳语（20dB）

#### D2 — 轻声细语（30 - 45 dB）

> 你的哈气响度较低，像安静环境下的猫咪哈气。对方如果离得近能听到，但远一点就听不清了。这种响度的哈气更像是一种"私人不满"，只有近距离的人才能感受到。

**参考对比：** 安静的办公室（40dB）、正常交谈的背景音（45dB）

#### D3 — 正常哈气（45 - 60 dB）

> 你的哈气响度处于标准区间，和日常对话差不多大。这是最常见的猫咪哈气响度范围，对方能清楚听到你的不满。在哈气PK中，这是"正常伤害"水平。

**参考对比：** 正常对话（60dB）、繁忙的餐厅（55dB）

#### D4 — 大声警告（60 - 75 dB）

> 你的哈气响度偏高，像吵闹环境下的猫咪哈气。这种响度已经能引起周围人的注意，传达的信息是"我很认真"。在哈气PK中，这是"高伤害"水平。

**参考对比：** 吸尘器（70dB）、繁忙的街道（65dB）

#### D5 — 震耳欲聋（75 - 100 dB）

> 你的哈气响度极高，像暴怒状态下的猫咪嘶吼。这种响度已经接近"疼痛阈值"，长时间暴露可能对听力造成影响。在哈气PK中，这是"暴击伤害"水平。你的哈气已经不是在表达不满了，是在发动声波攻击。

**参考对比：** 摩托车（85dB）、摇滚音乐会（100dB）

### 3.3 响度得分计算公式

```
dbScore = clamp((avgDB - 20) / 80 * 10, 0, 10)

其中：
- avgDB: 平均响度 (dB)
- 20: 起始响度偏移（低于20dB得0分）
- 80: 响度范围跨度（20dB到100dB映射到0-10分）
- clamp: 将结果限制在0-10范围内
```

**示例计算：**
| avgDB | 计算过程 | dbScore |
|-------|----------|---------|
| 15 dB | (15-20)/80*10 = -0.63 → 0 | 0 |
| 30 dB | (30-20)/80*10 = 1.25 | 1.3 |
| 45 dB | (45-20)/80*10 = 3.13 | 3.1 |
| 60 dB | (60-20)/80*10 = 5.0 | 5.0 |
| 75 dB | (75-20)/80*10 = 6.88 | 6.9 |
| 90 dB | (90-20)/80*10 = 8.75 | 8.8 |
| 100 dB | (100-20)/80*10 = 10.0 | 10.0 |

## 四、持续时间量化标准

### 4.1 持续时间等级划分

| 等级 | 名称 | 有效时长范围 | 猫咪行为类比 | 得分范围 | 视觉颜色 |
|------|------|------------|-------------|----------|----------|
| T1 | 瞬间哈气 | < 0.5 s | 打喷嚏式哈气 | 1-2分 | 白色 #FFFFFF |
| T2 | 短促哈气 | 0.5 - 1.0 s | 标准短哈气 | 3-4分 | 浅蓝 #87CEEB |
| T3 | 标准哈气 | 1.0 - 1.5 s | 典型猫咪哈气 | 5-6分 | 绿色 #32CD32 |
| T4 | 持久哈气 | 1.5 - 2.5 s | 耐力型哈气 | 7-8分 | 橙色 #FF8C00 |
| T5 | 超长哈气 | > 2.5 s | 传说级持久 | 9-10分 | 红色 #FF0000 |

### 4.2 持续时间等级详细描述

#### T1 — 瞬间哈气（< 0.5 s）

> 你的哈气转瞬即逝，像猫咪打了个喷嚏。对方可能还没反应过来你就哈完了。这种哈气的特点是"快"——出手快、收手也快，像闪电一样来去无踪。

**猫咪行为参考：** 猫咪被突然吓到时的短促哈气、快速警告后立即撤退。

#### T2 — 短促哈气（0.5 - 1.0 s）

> 你的哈气时间较短，但已经能完整传达"不满"的信息。这是最常见的猫咪哈气时长，通常伴随着一次明确的警告。对方能清楚地听到并理解你的意思。

**猫咪行为参考：** 猫咪看到陌生人时的标准短哈气、被碰了一下后的抗议哈气。

#### T3 — 标准哈气（1.0 - 1.5 s）

> 你的哈气时长处于标准偏长区间，说明你的肺活量和决心都超过了平均水平。这种时长的哈气通常意味着猫咪"真的很不高兴"，不是随便哈一下就完了。

**猫咪行为参考：** 猫咪被持续打扰时的哈气、对入侵领地者的正式警告。

#### T4 — 持久哈气（1.5 - 2.5 s）

> 你的哈气时长已经超过了大多数猫咪的平均水平。能持续哈气1.5秒以上的猫，说明它要么非常愤怒，要么肺活量惊人。在哈气PK中，持久型选手的优势在于"总伤害高"。

**猫咪行为参考：** 猫咪被逼到角落时的持续哈气、面对多个威胁时的防御性哈气。

#### T5 — 超长哈气（> 2.5 s）

> 你的哈气时长已经进入了"传说"级别。能持续哈气2.5秒以上，说明你的肺活量已经超越了99%的人类（和大多数猫咪）。这种时长的哈气在自然界中极为罕见，通常只有在极度愤怒或极度恐惧时才会出现。

**猫咪行为参考：** 猫咪暴怒状态下的超长嘶哈、面对生死威胁时的绝望哈气。

### 4.3 持续时间得分计算公式

```
durationScore = clamp((activeDuration - 0.3) / 2.7 * 10, 0, 10)

其中：
- activeDuration: 有效哈气时长 (s)，去除首尾静音
- 0.3: 起始时间偏移（低于0.3s得0分）
- 2.7: 时间范围跨度（0.3s到3.0s映射到0-10分）
- clamp: 将结果限制在0-10范围内
```

**示例计算：**
| activeDuration | 计算过程 | durationScore |
|---------------|----------|---------------|
| 0.2 s | (0.2-0.3)/2.7*10 = -0.37 → 0 | 0 |
| 0.5 s | (0.5-0.3)/2.7*10 = 0.74 | 0.7 |
| 1.0 s | (1.0-0.3)/2.7*10 = 2.59 | 2.6 |
| 1.5 s | (1.5-0.3)/2.7*10 = 4.44 | 4.4 |
| 2.0 s | (2.0-0.3)/2.7*10 = 6.30 | 6.3 |
| 2.5 s | (2.5-0.3)/2.7*10 = 8.15 | 8.1 |
| 3.0 s | (3.0-0.3)/2.7*10 = 10.0 | 10.0 |

## 五、混沌度量化标准

### 5.1 混沌度等级划分

| 等级 | 名称 | 频率方差范围 | 猫咪行为类比 | 得分范围 |
|------|------|-------------|-------------|----------|
| C1 | 稳如磐石 | < 300 Hz² | 像节拍器一样稳定 | 1-2分 |
| C2 | 轻微波动 | 300 - 600 Hz² | 偶尔抖一下 | 3-4分 |
| C3 | 正常变化 | 600 - 1000 Hz² | 标准猫咪哈气 | 5-6分 |
| C4 | 明显混乱 | 1000 - 2000 Hz² | 情绪波动较大 | 7-8分 |
| C5 | 彻底混沌 | > 2000 Hz² | 完全不可预测 | 9-10分 |

### 5.2 混沌度得分计算公式

```
chaosScore = clamp(freqVariance / 3000 * 10, 0, 10)

其中：
- freqVariance: 频率方差 (Hz²)
- 3000: 混沌度上限参考值
- clamp: 将结果限制在0-10范围内
```

## 六、综合评分算法（Audio_HIS）

### 6.1 HIS 公式

基于论文中的 HIS (Hissing Intensity Scale) 公式简化而来：

**原始公式：** `HIS = 0.4 * I_sound + 0.35 * θ_mouth + 0.25 * FI`

**音频简化版：**

```
Audio_HIS = 0.40 * dbScore + 0.25 * freqScore + 0.20 * durationScore + 0.15 * chaosScore
```

### 6.2 权重说明

| 维度 | 权重 | 依据 | 说明 |
|------|------|------|------|
| 响度 (dbScore) | 40% | 论文特征重要性排名 #1（权重0.28）+ 实际体验权重提升 | 哈气最直观的感受就是"多大声" |
| 频率 (freqScore) | 25% | 论文特征重要性排名 #3（权重0.19）+ 音调变化率（权重0.04） | 频率高低影响哈气的"尖锐程度" |
| 持久 (durationScore) | 20% | 论文特征重要性排名 #2（权重0.22） | 持续时间反映哈气的"决心" |
| 混沌 (chaosScore) | 15% | 综合其他特征（嘴部角度、毛发蓬松度等） | 混沌度反映哈气的"不可预测性" |

## 八、AI 对比打分规则

### 8.1 模板对比评分

#### 评分算法

```
对每个维度计算差异分：
  diffDB    = |userDB - templateDB| / 100
  diffFreq  = |userFreq - templateFreq| / 10000
  diffDur   = |userDur - templateDur| / 3.0
  diffChaos = |userChaos - templateChaos| / 3000

加权综合差异：
  totalDiff = 0.35 * diffDB + 0.30 * diffFreq + 0.20 * diffDur + 0.15 * diffChaos

相似度得分：
  similarity = (1 - totalDiff) * 100  // 范围 0-100

超越判定（加分项）：
  if userDB > templateDB && userFreq > templateFreq:
    bonusScore = min((userDB - templateDB)/10 + (userFreq - templateFreq)/1000, 20)
    similarity = min(similarity + bonusScore, 100)
```
