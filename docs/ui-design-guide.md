# 哈基米 UI 精简设计指引

> 来源：`06-UI设计文档.md` 精简版。用于当前分支的 UI 改造和后续页面实现。

## 设计定位

哈基米是一个手机优先的趣味音频测试。界面应该像一个轻松的小测试，而不是严肃工具。

核心关键词：

```text
趣味 / 暖色 / 猫系 / 直接可玩 / 方便分享
```

## 视觉原则

1. 第一屏直接可操作：用户打开后能立刻听示例、录音或上传。
2. 结果页要像结算卡：大分数、称号、猫咪素材、评分理由、分享按钮都清楚。
3. 文案像朋友聊天：少解释功能，多给用户行动提示和结果反馈。
4. 移动优先：主要内容宽度收敛，按钮足够大，文字不溢出。
5. 视觉轻快：使用橙、黄、粉、青绿作为重点色，避免冷硬科技感。

## 色彩

```css
--color-primary: #ff6b35;  /* 橘猫橙，用于主按钮和重点 */
--color-accent: #ffd166;   /* 阳光黄，用于奖励和提示 */
--color-pop: #ef476f;      /* 玫瑰红，用于高能反馈 */
--color-info: #118ab2;     /* 清爽蓝，用于解释信息 */
--color-success: #06a77d;  /* 成功绿 */
--color-bg: #f6fbf7;       /* 清爽浅底，避免厚重奶油色 */
--color-surface: #ffffff;  /* 内容面 */
--color-text: #2d2016;     /* 深棕文字 */
--color-muted: #735f4a;    /* 次级文字 */
--color-border: #eadfd4;   /* 轻边框 */
```

使用规则：

- 主 CTA 使用橘猫橙渐变。
- 结果分数和称号可使用玫瑰红或橘猫橙。
- 评分理由使用清爽蓝/浅青绿底，和主操作区区分。
- 页面整体不做单色铺满，颜色只服务于操作和反馈。

## 排版

系统字体优先，不加载外部字体。

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
  "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;
```

字号建议：

```text
主标题：40px 桌面 / 32px 手机
卡片标题：22px
正文：16px
辅助文字：14px
大分数：64px 桌面 / 48px 手机
```

## 组件

- 按钮高度不低于 48px。
- 主按钮：橘猫橙渐变、白字、明显阴影。
- 次按钮：白底、橘色边框或浅底。
- 卡片和功能面板圆角统一不超过 8px。
- 音频示例区域不要像二级卡片，做成轻量提示带。
- 评分理由用列表展示，最多 4 条，优先展示扣分点。

## 页面结构

入口页：

```text
Logo/猫咪素材
标题与一句提示
录音/上传主操作
示例音频播放
检测状态和预览
```

结果页：

```text
猫咪素材
认证徽章
相似度大分数
称号
评价文案
评分理由
分享文案
分享 / 再测一次
```

## 动效

- 录音中波形柱轻快跳动。
- 分数环和结果页切换使用 300ms 左右的过渡。
- 按钮按下有轻微反馈。
- 尊重 `prefers-reduced-motion`，减少动画。

## 无障碍

- 可点击元素至少 44px 高。
- 文本与背景保持清晰对比。
- 音频控件保留原生 controls。
- 重要状态用文字展示，不只依赖颜色。
