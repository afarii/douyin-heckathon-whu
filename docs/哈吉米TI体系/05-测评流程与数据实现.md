# 05 测评流程与数据实现

## 1. 数据链路

1. 埋点采集：播放、停留、点赞、评论、分享、收藏、二创。
2. ETL清洗：去重、反作弊、设备聚合、用户合并。
3. 评分引擎：内容TI评分 + MBTI问卷/行为融合。
4. 标签写回：写入用户标签与内容标签表。
5. 下游消费：推荐、活动、看板、消息触达。

## 2. 表结构建议

### 2.1 内容行为表 `dwd_content_action_di`

| 字段 | 类型 | 说明 |

| 字段 | 类型 | 说明 |
|---|---|---|

| calc_time | timestamp | 计算时间 |

## 3. 关键指标口径

- 分享率：`share_uv / play_uv`
- 收藏率：`save_uv / play_uv`
- 二创率：`remix_cnt / play_uv`
- 深度互动率：`(comment_uv + save_uv + remix_cnt) / play_uv`

## 4. 计算伪代码

```python
def calc_ti_level(a, c, v, b):
    score = round((a * 0.30 + c * 0.25 + v * 0.25 + b * 0.20) * 20)
    if score >= 90:
        return score, "TI-S"
    if score >= 80:
        return score, "TI-A"
    if score >= 70:
        return score, "TI-B"
    if score >= 60:
        return score, "TI-C"
    if score >= 50:
        return score, "TI-D"
    return score, "TI-F"
```

## 5. SQL样例（用户标签写回）

```sql
insert overwrite table ads_ti_user_tag_df
select
  u.user_id,
  t.ti_level,
  m.mbti_type,
  r.title_level,
  s.user_segment,
  'ti_v2_2026q2' as calc_version,
  current_timestamp() as calc_time
from tmp_ti_level t
join tmp_mbti_result m on t.user_id = m.user_id
join tmp_title_result r on t.user_id = r.user_id
join tmp_segment_result s on t.user_id = s.user_id
join dim_user u on t.user_id = u.user_id;
```

## 6. 调度与SLA

- 日任务：内容评分（T+1 08:00前完成）。
- 周任务：用户MBTI修正与分层更新（周一10:00前）。
- 月任务：称号升降级批处理（每月1日12:00前）。

## 7. 质量控制

- 空值率阈值：核心字段空值率<1%。
- 波动阈值：关键分层比例周环比波动>20%触发告警。
- 对账机制：结果表记录数需与用户基表对齐误差<0.5%。
