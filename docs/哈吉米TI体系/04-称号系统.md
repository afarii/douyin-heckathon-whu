# 04 称号系统

## 1. 设计目标

- 给用户明确成长路径与里程碑反馈。
- 强化社群认同，提升长期留存。
- 对创作、传播、治理贡献进行可见激励。

## 2. 命名结构

`[稀有度前缀] [TI等级] · [核心定位] · [文化标签] · [后缀]`

示例：`💎 TI-S · 圆哈镇之主 · 抽象大师 · 耄耋`

## 3. 稀有度规则

| 稀有度 | 比例建议 | 获取方式 |
|---|---:|---|
| SSR | 0.5% | 年度级贡献、重大创作里程碑 |
| SR | 3% | 季度级高质量贡献 |
| R | 12% | 连续稳定贡献 |
| N | 84.5% | 常规参与获取 |

## 4. 发放条件模板

| 类型 | 样例条件 |
|---|---|
| 创作类 | 30天内原创>=5且平均互动>=阈值 |
| 传播类 | 30天内有效分享>=100且转化达标 |
| 社区类 | 组织活动>=2且满意度>=4.5/5 |
| 守护类 | 风险内容举报命中>=5 |

## 5. 升降级机制

- 升级窗口：每月一次，按近30天滚动计算。
- 降级窗口：连续60天不达标触发降一级。
- 豁免条件：核心贡献者在维护期可暂缓一次降级。

## 6. 回收与申诉

- 回收场景：违规内容、作弊刷量、恶意攻击。
- 回收流程：系统命中 -> 人工复核 -> 通知 -> 生效。
- 申诉周期：7日内可提交证据并复审一次。

## 7. 称号词库维护

- 每月新增词条走评审（运营+内容+治理）。
- 词条要求：正向表达、可解释、避免歧义。
- 历史称号保留映射，不直接硬删除。
